Problem

Given a map image with colored markers (A, B, C) and red path boundary lines, the workflow extracts the marker centroids and line equations, then drives a robot along the computed path and finally actuates a manipulator to reach the target region. :contentReference[oaicite:11]{index=11}

Input map: markers and path to be extracted.

Marker extraction

  • Color classification in RGB with a purity coefficient δ to isolate blue markers (A, B) and green marker (C). :contentReference[oaicite:12]{index=12}
  • Morphological erosion + dilation using a circular structuring element sized to the marker radius to remove noise. :contentReference[oaicite:13]{index=13}
  • Separate the two blue markers by perimeter detection and connected-perimeter labeling; compute centroids via image moments. :contentReference[oaicite:14]{index=14}
Marker extraction stages: classification and morphology.

Path boundary extraction

  • Color classification to isolate near-pure red path lines. :contentReference[oaicite:15]{index=15}
  • Edge detection via convolution filters for horizontal/vertical edges and binarization to avoid “red blobs” becoming lines. :contentReference[oaicite:16]{index=16}
  • Hough transform (ρ, θ) to find dominant lines; reconstruct line elements and use known marker positions to identify the usable path. :contentReference[oaicite:17]{index=17}
Line detection: edges → Hough peaks → line elements.

Results (vision)

The report lists extracted marker centroids and line equations (four inner lines used as path boundaries), providing the geometric basis for motion planning. :contentReference[oaicite:18]{index=18}

Extracted centroids and path boundary line equations.

Mobile robot kinematics

The robot is modeled as a differential drive platform; wheel speeds are computed from desired body velocity using inverse kinematics. The planned motion is segmented into straight–turn–straight with chosen parameters (example values include v, ω, R, and axle half-length). :contentReference[oaicite:19]{index=19}

Differential-drive model and wheel-rate relationships.

Manipulator actuation

A desktop 4-DOF arm mounted on the mobile platform is used to reach the target region after the base moves into position; the task is simplified to placing the end-effector within the green marker region at some altitude, with a noted strategy of continuing past marker B to reduce the reach requirement. :contentReference[oaicite:20]{index=20}
