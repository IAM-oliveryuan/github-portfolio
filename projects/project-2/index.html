Desktop Robot Arm

A 4-DOF desktop manipulator (3 revolute + 1 prismatic) designed and analyzed via DH parameters, forward/inverse kinematics, Jacobian derivation, and joint-space trajectory planning.

Source Demo (optional)

Overview

The arm is intended for desktop-scale tasks (moving objects, holding tools, providing lighting) and is modeled using standard robotics tooling: DH parameters, transformation matrices, and velocity mapping through a Jacobian. :contentReference[oaicite:2]{index=2}

Mechanical design + DH model

The design uses three consecutive revolute joints and a prismatic joint. A DH table is extracted from the sketch, and fixed link lengths are chosen to avoid collisions with the desktop plane. :contentReference[oaicite:3]{index=3}

Sketch + frame assignment used to read DH parameters.

Forward / inverse kinematics

Forward kinematics is obtained by multiplying the chain transforms to produce the end-effector pose. Inverse kinematics is solved procedurally for (θ1, θ2, θ3, d4), with feasibility depending on whether the requested pose is reachable with only 4 independent variables. :contentReference[oaicite:4]{index=4}

Workspace visualization (point-sampled) from forward kinematics.

Jacobian derivation

The Jacobian is derived from linear and angular velocity propagation across frames, producing a mapping between joint rates and end-effector velocity. :contentReference[oaicite:5]{index=5}

Systematic Jacobian derivation using frame velocities.

Trajectory planning (joint space)

Via points are selected in (θ1, θ2, θ3, d4). Interpolation uses piecewise quadratic segments for variables with repeated values (θ2, d4) and piecewise cubic spline for continuously varying variables (θ1, θ3), with zero start/end velocity constraints. :contentReference[oaicite:6]{index=6}

Joint-space trajectories and smoothness constraints.

Results and discussion

  • Workspace coverage: achieved via full rotation of the first revolute joint and vertical range via joint 2 + prismatic motion. :contentReference[oaicite:7]{index=7}
  • Motion safety: velocity/acceleration magnitudes discussed, with a note that reducing δt improves efficiency but increases rates. :contentReference[oaicite:8]{index=8}
  • Future improvement: reduce sharp turns in acceleration by using higher-order piecewise polynomials. :contentReference[oaicite:9]{index=9}
